# 업무 자동화 정리

## 목적

반복적이고 규칙이 명확한 업무를 자동화해서 다음을 줄이는 것이 목표다.

- 수작업 입력
- 누락과 오입력
- 진행 상황 확인 비용
- 보고서 작성 시간
- 담당자 의존도

## 자동화하기 좋은 업무 기준

아래 조건에 해당하면 자동화 우선순위가 높다.

- 매일, 매주, 매월 반복된다
- 입력 형식이 정해져 있다
- 사람이 해도 판단 기준이 거의 같다
- 여러 시스템에 같은 내용을 반복 입력한다
- 마감 알림, 상태 확인, 취합 같은 추적 업무가 많다
- 실수 비용은 크지만 로직은 단순하다

## 자동화 가능한 업무 목록

### 1. 데이터 수집 및 취합

- 이메일, 메신저, 설문, 엑셀, CSV에서 데이터 자동 수집
- 여러 시트나 파일의 데이터 병합
- 정해진 형식으로 컬럼 정리 및 전처리
- 일별, 주별 실적 데이터 자동 집계
- 외부 시스템 API 연동으로 정보 자동 동기화

예시:
- 영업 실적 파일을 매일 모아서 하나의 대시보드로 합치기
- 거래처별 주문 데이터를 자동 취합해 담당자에게 전달하기

### 2. 보고서 및 문서 생성

- 주간 보고, 월간 보고 자동 초안 작성
- 실적표 기반 요약 문장 자동 생성
- 회의록 템플릿 자동 생성
- 견적서, 발주서, 확인서 같은 반복 문서 자동 작성
- 문서 파일명, 버전, 날짜 규칙 자동 관리

예시:
- 엑셀 수치를 읽어 주간 보고서를 Markdown, PDF, 이메일 본문으로 생성
- 회의 메모를 정리해서 액션 아이템 중심 회의록으로 변환

### 3. 승인 및 요청 프로세스

- 휴가, 지출, 구매 요청 접수 후 자동 라우팅
- 승인 상태 변경 시 담당자 자동 알림
- 누락된 승인 건 리마인드 발송
- 승인 완료 후 후속 작업 자동 실행

예시:
- 구매 요청서 제출 시 팀장 승인 요청 메시지 발송
- 승인 완료 후 회계 시트와 비용 현황 자동 반영

### 4. 일정 및 알림 관리

- 마감일 기준 자동 알림 발송
- 미완료 작업 자동 추적
- 일정 변경 시 관련자 일괄 통지
- 정기 업무 시작/종료 체크 자동화

예시:
- 매주 금요일 16시에 미제출 보고 대상자에게 자동 알림
- 계약 만료 30일 전 고객사 담당자와 내부 운영팀에 사전 안내

### 5. 고객 및 운영 대응

- 자주 묻는 질문 자동 응답
- 문의 내용 분류 후 담당 부서 자동 배정
- 접수 번호 자동 발급
- 처리 상태 변경 시 고객 안내 메시지 자동 발송

예시:
- 고객 문의를 배송, 환불, 기술지원으로 자동 분류
- 반복 문의는 템플릿 답변으로 1차 처리

### 6. 재무 및 정산 지원

- 거래 내역 정리 및 계정별 분류
- 세금계산서, 영수증, 정산 자료 취합
- 월말 정산 체크리스트 자동 생성
- 미수금, 미지급금 현황 자동 알림

예시:
- 카드 사용 내역을 읽어 비용 항목별로 분류
- 세금계산서 수신 여부를 자동 점검해 누락 건 알림

### 7. 인사 및 총무 업무

- 입사/퇴사 체크리스트 자동 생성
- 온보딩 문서 및 계정 발급 요청 자동화
- 근태 데이터 취합 및 이상 내역 표시
- 자산 반납, 좌석 배정, 비품 신청 자동 처리

예시:
- 신규 입사자 등록 시 장비, 계정, 교육 일정 생성
- 퇴사 일정 등록 시 접근 권한 회수 작업 자동 생성

### 8. 개발 및 IT 운영 업무

- 정기 배포 체크리스트 자동화
- 로그 수집 및 장애 알림
- 서버 상태 점검 보고 자동 생성
- 백업, 배치, 파일 정리 작업 스케줄링
- 코드 리뷰 보조, 문서 생성, 테스트 실행 자동화

예시:
- 장애 로그를 감지하면 슬랙 알림과 담당자 티켓 생성
- 릴리스 노트를 커밋 내역 기준으로 자동 초안 작성

## 우선 도입 추천 업무

작게 시작하려면 아래 5개가 효율이 높다.

1. 반복 보고서 자동 생성
2. 마감 및 미처리 건 알림 자동화
3. 엑셀/CSV 취합 자동화
4. 승인 요청 및 상태 알림 자동화
5. 반복 문의 템플릿 응답 자동화

## 자동화 도입 절차

1. 반복 업무를 목록으로 정리한다.
2. 건당 처리 시간, 빈도, 오류율을 기록한다.
3. 규칙이 명확한 업무부터 우선 선정한다.
4. 입력값, 처리 로직, 결과물을 정의한다.
5. 작은 범위로 먼저 자동화하고 검증한다.
6. 예외 처리와 수동 개입 지점을 문서화한다.
7. 운영 지표를 보고 점진적으로 확대한다.

## 자동화 대상 선정 체크리스트

- 담당자가 바뀌어도 같은 방식으로 처리할 수 있는가
- 입력 데이터가 디지털 형태로 존재하는가
- 결과물이 템플릿화 가능한가
- 예외 케이스가 전체의 일부인가
- 자동화 후 검수 기준을 만들 수 있는가
- 실패했을 때 수동 복구가 가능한가

## 기대 효과

- 처리 시간 단축
- 누락 및 오입력 감소
- 업무 진행 가시성 향상
- 담당자 의존도 감소
- 보고 품질 표준화
- 핵심 업무에 더 많은 시간 투입 가능

## 다음 단계 제안

바로 시작하려면 다음 형식으로 자동화 후보를 정리하면 된다.

| 업무명 | 현재 방식 | 반복 주기 | 소요 시간 | 오류/누락 위험 | 자동화 방식 |
| --- | --- | --- | --- | --- | --- |
| 주간 보고 작성 | 엑셀 수작업 정리 | 매주 | 2시간 | 중간 | 데이터 취합 + 보고서 자동 생성 |
| 승인 요청 알림 | 메신저 수동 전달 | 수시 | 20분 | 높음 | 제출 시 자동 알림 |
| 문의 분류 | 담당자 수동 확인 | 매일 | 1시간 | 중간 | 규칙 기반 자동 분류 |

이 문서를 기준으로 실제 팀 업무를 넣으면 자동화 우선순위 표로 바로 확장할 수 있다.
